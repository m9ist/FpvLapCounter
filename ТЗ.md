# ТЗ: FPV Lap Counter

## Контекст

Анализ видеозаписей FPV-полётов в помещении. Трасса включает стартовые/финишные ворота
(большая синяя LED-арка) и трассовые ворота (маленькие разноцветные LED-рамки).
Задача: автоматически найти все пролёты через стартовые ворота, посчитать времена кругов,
найти лучшие N кругов подряд.

---

## Входные данные

- Папка с видеофайлами формата `.ts` (основной) + MP4/AVI/MOV
- 1–5 референсных скриншотов стартовых ворот (JPEG/PNG)
- OSD-оверлей на видео: таймер полёта, формат `MM:SS.s`

### Особенности съёмки

На тренировке одновременно могут летать 2–4 дрона с разными камерами:
- Разное качество и разрешение изображения
- Разное расположение OSD-таймера на экране
- Возможна разная цветопередача (влияет на референсы)

Следствия для приложения:
- OSD-регион настраивается вручную на drag-and-drop кроп, отдельно для каждого видео (или копируется из предыдущего)
- Референсные кадры можно назначать на группу видео или на каждое видео отдельно
- При несовпадении камеры система предупреждает, если сходство с референсами аномально низкое

## Функциональность

### 1. Менеджер видео (папка)

- Пользователь выбирает папку (PowerShell dialog)
- Приложение сканирует папку и показывает список `.ts` файлов
- У каждого файла:
  - Чекбокс "обработать"
  - Статус: `⬜ не обработан` / `🔄 в очереди` / `✅ готов` / `⚠️ нет кругов`
  - Если рядом с файлом уже есть `имя_файла.fpv.json` — статус `✅ готов`, повторная обработка не нужна
- Кнопка "Выбрать все / снять все"
- Кнопка "🚀 Обработать выбранные" — запускает последовательную обработку
- Клик на файл — открывает его результаты в главной области
- Группировка видео по дрону/камере (опционально, через тег в JSON)

### 2. Детекция ворот — нейросеть

- Выбор модели из списка доступных (см. раздел "Модели")
- Модель считает косинусное сходство кадров видео с референсами
- Временной ряд сглаживается → пики = кандидаты на пролёты
- Параметры: порог сходства, мин. время круга, каждый N-й кадр, выраженность пика

#### Управление референсными кадрами

Три способа добавить референс:

1. **Файл** — кнопка `[ + Добавить файл ]`, выбор JPEG/PNG
2. **Буфер обмена** — кнопка `[ 📋 Вставить из буфера ]` (Ctrl+V или кнопка),
   считывает изображение из clipboard через `PIL.ImageGrab`
3. **Текущий кадр видео** — кнопка `[ 🎯 Использовать этот кадр ]` в блоке просмотра,
   добавляет текущий отображаемый кадр как референс

Референсы отображаются миниатюрами с кнопкой `×` для удаления.
Референсы сохраняются в `.fpv.json` (как base64) и восстанавливаются при следующем открытии.

#### Назначение референсов

- **Общие** — применяются ко всем видео в папке (по умолчанию)
- **На видео** — можно переопределить референсы для конкретного файла
  (кнопка "⚙️ Свои референсы для этого видео")
- Полезно когда в папке видео с разных дронов/камер

### 3. Ручная верификация пролётов

- После автодетекции показывается сетка кадров-кандидатов:
  ```
  [★ кадр 1]  [★ кадр 2]  [★ кадр 3]
   t=23.40s    t=53.66s    t=86.65s
   sim=0.891   sim=0.876   sim=0.903
  [✅ верно]  [❌ фейк]   [✅ верно]
  ```
- Кнопки ✅/❌ под каждым кадром — пометить как реальный пролёт или фейк
- Фейки исключаются из расчёта кругов
- Статус верификации сохраняется в JSON

### 4. Считывание OSD-таймера

- EasyOCR читает таймер только с кадров-кандидатов (экономия времени)
- Время пролёта = OSD-время; fallback: frame_index / fps
- Регион OCR: настраивается вручную через drag-and-drop кроп на превью кадра
- Регион сохраняется в `.fpv.json` и применяется ко всем кадрам этого видео
- Можно скопировать регион с одного видео на другое (у дронов одной камеры регион одинаковый)

### 5. Расчёт кругов

- Lap time = разница OSD-времён между соседними подтверждёнными пролётами
- Фильтр минимального времени круга
- Фильтр регулярности: отклонение ≤ X% от медианы (опционально)

### 6. Лучшие N кругов подряд

- Скользящее окно по массиву кругов
- Результат для N = 1, 3, 5 (настраивается пользователем)

### 7. Сохранение результатов

- Автосохранение в `<имя_видео>.fpv.json` рядом с видеофайлом после анализа
- Формат JSON:
  ```json
  {
    "video": "race1.ts",
    "model": "ViT-B-32",
    "references": ["gate1.jpg", "gate2.jpg"],
    "analyzed_at": "2025-01-15T14:32:00",
    "params": { "threshold": 0.75, "min_lap_sec": 25, "sample_every": 3 },
    "passes": [
      { "frame": 1404, "time_sec": 23.40, "osd_time": 20.3,
        "similarity": 0.891, "verified": true }
    ],
    "laps": [
      { "lap": 1, "duration_sec": 30.26, "start_sec": 23.40 }
    ],
    "best_lap": { "lap": 1, "duration_sec": 29.45 },
    "best_3_consecutive": { "laps": [1,2,3], "total_sec": 94.54 }
  }
  ```
- При повторном открытии — загружается из JSON, анализ не запускается
- Кнопка "♻️ Пересчитать" для принудительного повтора

### 8. Сравнение видео в папке

- Сводная таблица по всем обработанным файлам
- Сортировка по лучшему кругу / лучшим 3 подряд / среднему
- Экспорт в CSV

## Модели для детекции

Выбор в боковой панели. Приложение показывает только те, что можно запустить локально:

| Модель | Размер | Точность | Скорость (RTX 3060) |
|---|---|---|---|
| CLIP ViT-B/32 | 350 МБ | ★★★☆ | ~200 кадр/с |
| CLIP ViT-L/14 | 890 МБ | ★★★★ | ~80 кадр/с |
| CLIP ViT-H/14 | 2.5 ГБ | ★★★★★ | ~30 кадр/с |
| DINOv2 ViT-B/14 | 330 МБ | ★★★★ | ~150 кадр/с |
| DINOv2 ViT-L/14 | 1.1 ГБ | ★★★★★ | ~60 кадр/с |

- При первом выборе модель скачивается автоматически (~HuggingFace/OpenCLIP)
- Модель кешируется локально, повторная загрузка не нужна
- Активная модель отображается в заголовке

## Производительность

- Результаты кешируются в `.fpv.json` — повторный запуск мгновенный
- CLIP-модель: один инстанс на всю сессию
- Батч инференс: 16–32 кадра
- EasyOCR: только на кадрах-кандидатах (±2с вокруг пика)
- Прогресс-бар при обработке каждого файла

---

## Решения по архитектуре

- Группировка по дрону — не нужна, плоский список
- Референсы хранятся локально в папке с видео (`references/` или inline в `.fpv.json`)
- Только графический интерфейс (Streamlit), CLI не нужен
